🔄 Change Management Process

Network Engineer: Kagiso Nkalanga  |  June 2026  |  APPROVED

1. Change Management Overview

This document defines the formal process for managing changes to the Liya's Beauty Boutique network infrastructure, systems, and applications. All changes must follow this process to ensure stability, security, and proper documentation.

2. Change Types

🟢 Standard Changes Pre-approved, low-risk changes
• Password resets
• User account creation
• Software updates
• Backup verification
Approval: Network Engineer
🟡 Normal Changes Medium-risk, requires approval
• New software installation
• Hardware replacement
• Firewall rule updates
• Network configuration changes
Approval: Change Advisory Board
🔴 Emergency Changes High-risk, immediate action required
• Security breach response
• Critical system outage
• Data corruption recovery
• Disaster recovery execution
Approval: Emergency CAB

7. Emergency Change Process

Emergency changes bypass normal approval but must follow:

  1. Immediate action to resolve critical issue
  2. Notify Change Manager within 30 minutes
  3. Document change in emergency log
  4. Post-implementation review within 24 hours
  5. Retrospective approval by CAB
